# Contento — план улучшений до MVP (закрытая бета)

Дата: 2026-06-11
Статус: черновик на ревью

## Цель

Закрытая бета с реальными клиентами на RU-рынке: клиент проходит онбординг, система
стабильно производит вертикальные видео (9:16) с консистентным AI-аватаром, субтитрами
и брендированием, клиент апрувит, система публикует в VK / Telegram / Instagram Reels.

Стек генерации (зафиксировано): **Higgsfield (Soul + Speak) + ElevenLabs (TTS) + Remotion (монтаж)**.
Higgsfield вызывается по REST API из video-worker (как сейчас); MCP — только для ручных экспериментов.

## Диагноз (кратко)

Архитектура пайплайна собрана и работает (Campaign → ContentPlan → Script → VideoJob →
Publication, 6 реальных паблишеров, воркеры на BullMQ/Kafka). Плохое качество вывода
вызвано конкретными исправимыми причинами:

**Видео:**
1. `duration: 5` захардкожен в Speak-вызове (`packages/ai/src/higgsfield/client.ts:163`) — липсинк ломается на любой реплике ≠ 5 сек.
2. Самые дешёвые параметры: `quality: 'mid'` (Speak), модель `dop-lite` (image2video), кадры Soul в 1536×2048 (3:4, не 9:16).
3. Персонаж не консистентен: новый Soul-кадр на каждый шот по разным промптам; `characterDescription` из AvatarPersona не передаётся в storyboard (`apps/video-worker/src/worker.ts:84-88`); `soulId` берётся только из env, минуя персону из БД (`worker.ts:110`).
4. Нет negative prompts ни в одном вызове Higgsfield.
5. Монтаж: ffmpeg-concat без нормализации разрешения, переходов и субтитров; CRF 23.
6. Надёжность: нет ретраев к внешним API; таймаут поллинга 5 мин (client) vs 45 мин (campaign-producer); гонка webhook/polling может запустить stitch дважды.

**Тексты/сюжеты:**
1. System prompt scriptwriter — 3 строки, zero-shot, поле `platform` игнорируется, нет ограничений длины и структуры хука, temperature по умолчанию (1.0).
2. Язык вывода не зафиксирован нигде, кроме video-storyboard; в campaign-producer язык захардкожен `'ru'` (`campaign-producer.ts:141`).
3. Визуальные промпты storyboard «до 30 слов» без правил конкретики (камера/свет/действие/запрет текста в кадре).
4. buildBrandContext не содержит имени бренда, языка, платформ, ниши из CompanyPortrait.
5. Hook-формулы есть только в content-plan-generator; большинство агентов без Zod-валидации; storyboard-агенты в обход cost-трекера.

## План по фазам

### Фаза 0 — Стабилизация и quick wins (≈1 день)

Цель: текущие незакоммиченные фиксы доведены до коммита + дешёвые исправления с
максимальным эффектом на качество.

- Прогнать `pnpm typecheck` / `pnpm test`, закоммитить 18 изменённых файлов (фиксы аудио-загрузки на Higgsfield CDN, авторизованного скачивания клипов, Stop-гонок).
- **Длительность Speak = реальной длине аудио**: измерять длительность WAV (ffprobe или из заголовка PCM) и передавать её в `submitTalkingAvatarClip` вместо константы 5.
- `soulId` и `characterDescription` в worker брать из `AvatarPersona` (env — только fallback).
- `quality: 'high'` для Speak; вертикальный размер кадра Soul (9:16, напр. 1152×2048 — уточнить допустимые значения по докам Higgsfield); для image2video — полноценный `dop` вместо `dop-lite` + явная длительность.
- Единая константа negative prompt во все вызовы Higgsfield (артефакты рук, водяные знаки, текст в кадре, NSFW).
- Ретраи с экспоненциальным backoff (3 попытки) на все fetch к Higgsfield и ElevenLabs.
- Таймаут `pollJobUntilDone` 5 мин → 15 мин; идемпотентный stitch: атомарный переход `VideoJob` в `STITCHING` через условный `updateMany` (вторая попытка enqueue — no-op).

### Фаза 1 — Консистентность персонажа и качество картинки (2–4 дня)

- **Канонический образ**: один master-портрет персонажа на VideoJob (или `AvatarPersona.referenceImage`); все шоты генерируются с фиксированным текстовым блоком внешности (одежда, причёска, фон, свет), который добавляется префиксом к промпту каждого шота. Поднять `custom_reference_strength` (подобрать на тестах, ~0.9).
- **Переписать video-storyboard агент**: правила визуальной конкретики (субъект + действие + камера + свет, конкретные объекты, запрет абстракций и текста в кадре), стабильный style-block из Visual Identity бренда, few-shot пример хорошего шота, temperature ≈ 0.4.
- **Golden set**: скрипт `scripts/golden-videos.ts` — 3–5 фиксированных сценариев, по которым прогоняется пайплайн до/после изменений для визуального сравнения (и контроля стоимости за видео).

### Фаза 2 — Монтаж через Remotion (≈1 неделя)

Заменить ffmpeg-concat на Remotion-композицию (расширить `packages/brand-kit`):

- Canvas 1080×1920; клипы Higgsfield как `<OffthreadVideo>` слои с нормализацией scale/pad (решает падения concat на разных разрешениях).
- **Бёрн-ин субтитры** по `dialogue` шотов: тайминги слов из ElevenLabs (`with_timestamps` / alignment API). Субтитры критичны — short-form смотрят без звука.
- Переходы между шотами (короткий cross-fade / cut по биту), интро/аутро, CTA-экран.
- Брендирование из Brand Kit: цвет, шрифт, лого, плашка имени.
- Рендер: в video-worker через `@remotion/renderer` (или расширить render-worker); вывод 1080×1920 H.264.
- ffmpeg-concat остаётся фолбэком за фиче-флагом на время перехода.

### Фаза 3 — System prompts, RU-first (2–3 дня, параллелится с Фазой 2)

- **scriptwriter**: новый системный промпт — hook-формулы (перенести из content-plan-generator), платформенные ограничения (Reels ≤ 60–90 с, ~150 слов/60 с, VK Clips, TG), явная директива русского языка, требования «живой речи» (без канцелярита и AI-штампов), 2–3 few-shot примера, temperature 0.3–0.5.
- **idea-generator**: явная JSON-схема вывода в промпте, RU-директива.
- **brand-checker**: структурированный промпт с явными весами критериев.
- **buildBrandContext**: добавить имя бренда, язык воркспейса, целевые платформы, нишу/USP из CompanyPortrait.
- Язык: убрать хардкод `'ru'` в campaign-producer → настройка воркспейса (default `ru`).
- Инфраструктура агентов: Zod-схемы на вывод всех агентов; все вызовы через `runAnthropicMessage` (cost-tracking — сейчас оба storyboard-агента мимо него); явная temperature везде.
- Удалить мёртвых агентов (`storyboard.ts` legacy, `music-suggester`, `cover-concept` standalone, `viral-analyzer`) или пометить экспериментальными.
- Модели по стоимости: Haiku для trend/check-задач, Sonnet для генерации — как сейчас, зафиксировать как правило.

### Фаза 4 — Надёжность для беты (3–5 дней)

- Выбрать первичный механизм завершения шотов (рекомендация: polling в воркере; webhook — только ускоритель, через ту же идемпотентную функцию).
- Понятные ошибки в UI (VideoJobPanel): этап, на котором упало, и человекочитаемая причина.
- Учёт стоимости per video job (Higgsfield + ElevenLabs + Claude) — писать в БД/ClickHouse, показать в админке.
- E2E smoke-тест: онбординг → портрет → аватар → кампания → план → видео → ревью → публикация (на HIGGSFIELD_MOCK и на реальном API).
- Убрать дефолтные S3-креды из кода (`apps/video-worker/src/s3-client.ts:8-10`) — fail fast без env.

### Фаза 5 — Бета-полировка (≈1 неделя)

- Довести client review flow (страницы уже есть): нотификации клиенту, комментарии к отклонению, перегенерация по фидбеку.
- Паблишеры под RU-приоритет: проверить **видео**-публикацию VK (video upload) и Telegram (sendVideo), добавить Instagram Reels (`media_type=REELS`); TikTok/YouTube — уже умеют видео, вторым приоритетом.
- Минимальный дашборд статуса кампании (что в продакшене, что на ревью, что опубликовано). Полная аналитика — после MVP.

## Что сознательно НЕ входит в MVP

- Полный analytics-дашборд (ClickHouse-метрики остаются собираться).
- LoRA fine-tuning, A/B-тесты, mentions-мониторинг — есть в коде, не блокируют бету.
- Мультиязычность (EN — после RU-беты).
- Биллинг.

## Критерии приёмки MVP

1. 10 видео подряд по реальной кампании генерируются без ручного вмешательства; ≥ 8 из 10 проходят клиентское ревью без правок.
2. Видео: 1080×1920, один и тот же персонаж во всех кадрах, липсинк без обрезок/заморозок, бёрн-ин субтитры, брендовые цвета/шрифт.
3. Тексты: русский язык всегда, хук ≤ 3 сек, длина под платформу.
4. Публикация по расписанию в VK + Telegram + Instagram Reels.
5. Известна себестоимость одного видео (по job).
