# Per-platform strategy, lipsync A/B, Lambda — research synthesis & decisions

Дата: 2026-06-15. Источник: research-workflow (5 агентов, RU-рынок, 2026-актуальные источники). Все цифры — directional; платформенные алгоритмы меняются часто.

## 0. Headline: гео-реальность важнее любого формата

Главный вывод исследования — не про форматы, а про **доступность платформ в RU**. Это меняет саму постановку «RU-рынок»:

- **TikTok:** загрузка с российских IP **заблокирована** с марта 2022 (всё ещё в силе в 2026). Из России можно только смотреть. Публиковать реально только с **не-RU/CIS аккаунта**, целясь в русскоязычную диаспору + Центральную Азию (Казахстан, Узбекистан, Киргизия, Армения, Грузия, Молдова — там живые экосистемы). Беларусь — полный бан.
- **Instagram:** в России **заблокирован**, Meta признана экстремистской, **платное продвижение незаконно с 01.09.2025** (штрафы юрлицам 100–500k₽). Осталось ~16% аудитории через VPN и сокращается. Юридически рискованно таргетить мейнланд РФ.
- **YouTube:** в России замедлен с авг. 2024 (~42% охвата, в ~10× медленнее). Остался самоотобранный VPN-кор. **Rutube (~79%) и VK Video (~66%) имеют более широкий охват в самой РФ**, но слабее рекомендации.
- **Telegram:** работает везде, включая мейнланд РФ — единственный канал без гео-ограничений.

**Вывод/противоречие, которое надо решить пользователю:** TikTok + Instagram + YouTube Shorts по сути обслуживают **русскоязычную диаспору + СНГ**, а не мейнланд Россию. Решение «убрать VK» согласуется с диаспора/СНГ-стратегией, но **противоречит** стратегии «мейнланд РФ» (там реальный охват дают VK Video / Rutube / Telegram). Нужно зафиксировать целевой рынок до того, как вкладываться в per-platform логику (см. вопрос в конце).

## 1. Per-platform: ЦА, форматы, алгоритм (то, ради чего был research)

Общий базис (все три): вертикаль 9:16 1080×1920; **бёрн-ин субтитры обязательны** (~85% смотрят без звука); хук в первые 1–3 с; **completion rate — сигнал №1**. Это у нас уже закрыто Фазами 0/2. Дальше — различия, которые требуют **отдельной логики на платформу**:

### TikTok (диаспора + Центр. Азия)
- **ЦА:** 16–24 (~55%), женский перекос (~57%), терпимость к сырому/аутентичному, низкая — к «рекламному»/очевидному AI.
- **Форматы-победители:** короткие сериалы/мини-серии; storytime/POV; talking-head «как живой, не корпоратив»; **тренд-саунд скетчи/мемы**; листиклы с текст-оверлеем + b-roll. Туториалы слабее, если хук не переформулирован в «результат» в первые 3 с.
- **Уникальное:** **нативный/трендовый звук — высокий вес именно на TikTok** (не на Reels/Shorts). Длина 21–34 с (60–90 с растёт, если completion держится). Шеры-в-DM > комменты > лайки. **SEO/search — прямой сигнал.**
- **Анти:** видимые ватермарки чужих платформ (охват <100), дубли (perceptual-hash >85% → суппресс), корпоративный/уканный AI (−30–45%).

### Instagram Reels (диаспора + урбан-РФ через VPN)
- **ЦА:** 25–34 крупнейшая, женский перекос, урбан/выше доход, beauty/fashion/lifestyle/эстетика.
- **Форматы:** **UGC «говоришь как другу» face-to-cam** (топ); **b-roll + жирный текст-оверлей** (sound-off-first, смена кадра каждые ~1.5 с); листиклы (сасейвы ~3× лайка); before/after. **Story-time РИСКОВАН для AI-аватара** (синтетическую озвучку детектят).
- **Уникальное:** **sends (DM-шеры) — топ-сигнал, ×3–5 лайка**, главный драйвер охвата нефолловеров. Длина 7–15 с (loop) или 15–30 с. Подпись = SEO (индексируется текст + on-screen + транскрипт).
- **Анти:** ватермарки, репосты (−60–80%), слабые первые 3 с, очевидный низкокач AI, engagement-bait.

### YouTube Shorts (диаспора + VPN-кор РФ + СНГ)
- **ЦА:** 18–40, урбан, info/entertainment, технически способные (VPN). СНГ — шире.
- **Форматы:** **face-to-cam + E-E-A-T (человеческое присутствие) вознаграждается сильнее всех**; «фейс + текст»; туториалы/лайфхаки (search-discoverable); листиклы **только с реальным ведущим**.
- **Уникальное:** **«Viewed vs Swiped Away» в первую ~1 с** — гейт распределения; **search/SEO — реальный канал** (title + description + произнесённые слова в авто-транскрипте ранжируют). Длина 20–35 с.
- **Анти (критично для AI-фабрики):** **faceless mass-produced AI (сток + робо-войсовер, авто-«Топ-10») активно суппрессится (часто <1000 показов)**; репосты с ватермарком; misleading title.

**Сквозной вывод для AI-аватар-фабрики:** все три платформы (а Shorts — особенно жёстко) **давят очевидный/faceless AI**. Это валидирует решение #3 (multi-format) и поднимает планку реализма аватара: «один аватар везде» рискован. Нужна **per-platform адаптация формата, длины, подписи, доли аватар-vs-broll**.

## 2. Архитектура per-platform логики (предложение)

Ввести **`PlatformProfile`** — конфиг на платформу, который ведёт генерацию и упаковку. Поля (из research):
`targetDurationSec` (band), `hookWindowSec`, `preferredFormats` (ранжир: avatar / broll-text / screencast / serial), `avatarShare` (доля аватар-шотов vs b-roll), `nativeSoundImportance` (high для TikTok), `captionStyle` (SEO-keyword-first для Reels/Shorts; разговорный + тренд-тег для TikTok), `hashtagCount`, `subtitleStyle`, `aigcDisclosure: true`, `cadence`.

Профиль прокидывается в:
- **content-plan / scriptwriter / video-storyboard** — выбирают формат, длину, хук, долю аватар/broll под платформу (связка с decision #3 multi-format: добавить `shotType` в `VideoShotSchema` — `avatar` | `broll-text` | `screencast`).
- **Remotion-композицию** — длительность, стиль субтитров, на TikTok — слот под нативный звук.
- **caption-генерацию** — отдельный per-platform рерайт (сейчас одна подпись для всех + `platform:'instagram'` хардкод — это надо менять; см. P1 §2.9).
- **publisher** — `aigcDisclosure` флаг (P1) обязателен везде.

Один скрипт → N платформенных вариантов (формат/длина/подпись/маркировка разные). Это и есть «своя логика на каждую платформу».

## 3. AIGC-маркировка (правило для билда)

Все три требуют disclosure реалистичного синтетика. **Сам лейбл — НЕ штраф к ранжированию**, но **недекларированный** AI ретроактивно метится (C2PA / Google SynthID / аудио-вотермарки, в т.ч. ElevenLabs) → distribution hold / демонетизация. ElevenLabs-аудио несёт вотермарку, которую детектят — спрятать AI-происхождение всё равно нельзя. **Правило: всегда ставить AIGC-флаг в publish-пейлоаде** (TikTok AI-toggle, Meta AI-info, YouTube `containsSyntheticMedia`). Поднимает приоритет P1-задачи AIGC.

## 4. Decision #1 — lipsync A/B (Higgsfield vs HeyGen vs Sync.so)

| Провайдер | RU-липсинк | Вход | Макс клип | $/45с видео | Интеграция |
|-----------|-----------|------|-----------|-------------|------------|
| **Higgsfield Speak** (A, incumbent) | moderate-good (слабейший задокументирован) | **image+audio** (наш кейс) | 5/10/15 с (хард-кап) | ~$5.5 (annual) | готов |
| **HeyGen Avatar IV** (B, челленджер) | good, явный RU | **image+audio** (drop-in!) | 180 с/scene | **~$2.25–3.0 PAYG** | средняя |
| Sync.so | good, language-agnostic | **video→video** (НЕ image) | до 30 мин | ~$1.8–2.25 | **не drop-in** (рефайнер) |

**Рекомендация:** A = Higgsfield (уже в `VideoProvider`), **B = HeyGen Avatar IV** (тоже image+audio → ничего не ломает, дешевле PAYG, зрелее, лучший реализм 2026). **Sync.so — не замена** (нужен исходный видео-клип, не фото) — отложить как рефайнер. A/B вешается ровно на `VideoProvider`-интерфейс (P0 Task 10): добавить `HeyGenProvider`.

**A/B-протокол (пользователь гоняет сам — качество липсинка я оценить не могу):** один RU-скрипт ~45с / 6 шотов, включить шоты с трудными согласными («вздрогнул»), мягкими/щ/ж («счастье»), быстрый и медленный; озвучить ОДИН раз в ElevenLabs → переиспользовать те же WAV для обоих провайдеров; один Soul-кадр на всех. Рубрика оценки (1–5): синхрон губ на сложных кластерах, артефакты рта/зубов, естественность мимики, стабильность между шотами, $/видео. Я подготовлю стенд (HeyGenProvider + скрипт прогона) — оценку ставишь ты.

## 5. Decision #4 — Remotion Lambda (выбрано)

- **Стоимость:** ~$0.013–0.016 / 45с-видео (AWS). Биллинг = сумма всех параллельных lambda-секунд (конкурентность режет латентность, не стоимость).
- **Лицензия:** **Company License обязательна** (>3 сотрудников → платно; план «Automators» для автоматического рендера). Юр. обязательство — заложить.
- **Миграция (за флагом `VIDEO_STITCHER=lambda`, третий путь рядом с remotion/ffmpeg):** one-time AWS (`npx remotion lambda policies`), `deployFunction` (2048MB, disk 10240MB, timeout 240س) + `deploySite` (грузит `brand-kit/public` → шрифты/субтитры работают), `renderMediaOnLambda` + `getRenderProgress`-поллинг, затем `downloadMedia` → наш MinIO тем же ключом. `buildStitchProps`/`stitch-props.ts` переиспользуются как есть. Поднять concurrency воркера 2→10–25.
- **Блокеры (заложить):** (1) **клипы должны быть доступны Lambda по HTTPS** — реальный S3 или публичный MinIO; **из RU-хостинга egress в AWS может быть медленным/заблокированным** — выбрать регион/CDN; (2) presign-expiry поднять с 3600с (очередь + рендер могут пережить); (3) запросить увеличение AWS concurrency-квоты (новый аккаунт = 10); (4) версия Lambda пиннится к версии Remotion — при бампе re-deploy.

## Как это ложится в roadmap

- **P0** (видео доходит до публикации) — без изменений по сути; VK убран.
- **Новое перед/внутри Фазы 3:** `PlatformProfile` + per-platform логика + `shotType`/multi-format — это объединяет decision #3 и тему «каждая платформа уникальна». Крупный блок, тянет за собой scriptwriter/video-storyboard/Remotion/caption.
- **A/B липсинка:** добавить `HeyGenProvider` (после P0 Task 10) + стенд прогона.
- **Lambda:** третий путь стичера + AWS-обвязка + лицензия Remotin.
- **AIGC-маркировка** поднимается в приоритете (детектят всё равно).

## Открытый вопрос (блокирует объём)
Целевой рынок: **диаспора + СНГ** (TikTok/IG/Shorts + Telegram — то, что мы строим) **или мейнланд РФ** (тогда реальный охват — VK Video/Rutube/Telegram, а вложения в TikTok/IG почти не достигают РФ)? От этого зависит набор платформ и вся per-platform логика.
