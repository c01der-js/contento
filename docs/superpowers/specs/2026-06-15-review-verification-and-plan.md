# Архитектурное ревью — проверено по коду + исправленный план

Дата: 2026-06-15. Метод: 6 параллельных агентов прошли каждый `🔍 ПРОВЕРИТЬ В КОДЕ` маркер по реальным `.ts`-исходникам с `file:line`-доказательствами + веб-агент сверил внешние факты по первоисточникам. Каждый вердикт ниже подтверждён цитатой кода.

## Главный вывод (важнее, чем в исходном ревью)

Исходное ревью считало, что «публикует в соцсети» в основном работает и нужно лишь выбрать стратегию площадок. **Это не так. Сгенерированное видео вообще не доходит до публикации.**

- `PublishPayload` несёт только `text/imageUrl/hashtags` (`packages/platforms/src/types.ts:1-5`) — **поля видео нет**.
- Пейлоад собирается из `RenderJob.outputUrl`, а `RenderJob` производит **PNG-картинку** (`render-worker/src/worker.ts:117 renderStill`, ключ `...${renderJobId}.png`), не видео.
- В кампанийном пути `Publication` создаётся с `renderJobId: null` и **не имеет ссылки на `videoJobId`** (`campaigns.ts:296`). То есть `VideoJob.outputUrl` (готовый MP4 из Фазы 2) **никуда не передаётся**.
- Плюс **scheduled-публикация сломана**: scheduler не кладёт `platform` в BullMQ-джобу → Kafka-событие не проходит Zod-валидацию и молча теряется.

Итог: единственный рабочий end-to-end путь — немедленная публикация **PNG-картинки** через `social.ts`. Замены агентству нет не только из-за отсутствия контура обратной связи (это ревью угадало), но и потому, что **видео-продукт физически не публикуется**. Это и есть исправленный P0.

## Новые баги, найденные сверх ревью

1. **Видео не доходит до публикации** (P0) — нет связи `VideoJob.outputUrl → Publication`; пейлоад без поля видео; источник — PNG `RenderJob`.
2. **Scheduled-публикация молча теряется** (P0) — scheduler не передаёт `platform` → Kafka-событие не валидируется.
3. **`handleGenerate.createMany` не идемпотентен** (P1, латентный) — обязателен guard ДО включения ретраев BullMQ.
4. **`'x'`/Twitter в `SocialPlatformSchema` без паблишера** — `createPublisher('x')` кидает.
5. **`VideoJob.aspectRatio` — мёртвый столбец** — рекламирует конфигурируемость, которой нет.

## Внешние факты (поправки к ревью)

- **ElevenLabs Flash vs Turbo** ✅ — официально рекомендуют Flash во всех кейсах. Поправка: **цена одинаковая**, разница только в латентности (не дороже, не медленнее). Оба поддерживают `/with-timestamps`. → дешёвый свитч `turbo_v2_5 → flash_v2_5`.
- **ElevenLabs Music + SFX API** ✅ — оба существуют (Music — на платном плане). Музыкальный слой можно закрыть тем же вендором.
- **Sync.so** ✅ — API-first, usage-based ~$0.04–0.133/сек, есть Batch API и estimate-cost. Кандидат на программный липсинк.
- **HeyGen API** ✅ — зрелый API (~$0.05/сек, pay-as-you-go). Зрелее Higgsfield Speak для аватаров; Higgsfield сильнее в кинематографичности, но клипы ~14 с и слабее липсинк.
- **Remotion license** ✅ с поправкой: триггер — **только headcount, не выручка**. Для for-profit >3 сотрудников (4+) нужна Company License ~$25/dev/мес, минимум ~$100/мес. На 4+ людях это юридически обязательно.
- **TikTok API** — частично: аудит + SELF_ONLY до одобрения ✅; нативные звуки/стикеры/опросы только через draft-handoff ✅; но дневной лимит **~15/день, не 25** (ревью ошиблось); неаудированный клиент ≤5 пользователей/24ч.

## Исправленный план (сверён с уже сделанным)

Уже закрыто этой сессией: **Фаза 0** (quick-wins генерации) и **Фаза 2** (Remotion-монтаж: 9:16, субтитры, CTA, проверено реальным рендером). Поэтому ревью-фазы переразложены и приоритизированы заново:

### P0 — без этого продукта нет (сделать первым)
1. **Связать видео с публикацией.** `PublishPayload + videoUrl`; `Publication.videoJobId`; в approve-хендлере кампании заполнять из `item.videoJobId → VideoJob.outputUrl`; `posting-service` передаёт video URL.
2. **Включить видео в паблишерах:** IG Reels (`media_type:REELS`+`video_url`), Telegram `sendVideo`, VK `video.save`, LinkedIn video-asset. YouTube/TikTok уже умеют — дать им URL.
3. **Починить scheduled-путь** (`platform` в BullMQ-джобе) + интеграционный тест scheduled→Kafka→publish.
4. **`VideoProvider`-интерфейс** (`characterFrame/talkingHead/motionFromImage/poll`) + `HiggsfieldProvider`; переименовать vendor-поля в схеме/роутах; удалить мёртвый `video/higgsfield.ts`. Это разблокирует A/B Sync.so/HeyGen без переписывания воркера.

### P1 — надёжность и «работает в проде»
5. **BullMQ retries+backoff+removeOnFail/Complete**, но СНАЧАЛА сделать `handleGenerate` идемпотентным (guard на `createMany`). Подключить consumer для `publish.failed retryable`.
6. **QA-гейт перед CLIENT_REVIEW**: ffprobe-сверка длительности аудио vs клипа, клиппинг, субтитры в границах/safe-area.
7. **Частичная перегенерация шота** — эндпоинт `POST /video-jobs/:jobId/shots/:shotId/regenerate` + убрать «1 FAILED = весь job FAILED».
8. **Юнит-экономика per VideoJob**: добавить `videoJobId` в `llm_usage_events`; эмитить кост ElevenLabs (символы) / Higgsfield (джобы) / рендер (минуты).
9. **AIGC-маркировка** во всех пейлоадах (TikTok `brand_content_toggle`, Meta `ai_info`, YouTube `containsSyntheticMedia`) + workspace-тоггл.

### P2 — дифференциация («лучший на рынке», главный ров)
10. **Контур обратной связи**: воркер тянет insights (IG/TikTok/YT) → `PostMetrics`; агрегаты по формату/пиллару кормят `generateContentPlan` и промпты idea/script; `hooks-evolve` от реального engagement.
11. **Живые тренды в scoring и план**: прокинуть engagement-сигнал фетчеров в `analyzeTrend`; заполнять `virloTrends` в контент-план.
12. **Музыкальный слой** (ElevenLabs Music/SFX → `<Audio>` в VideoStitch) + трендовый звук.
13. **Разнообразие форматов**: `shotType` в `VideoShotSchema`; ветка без аватара (b-roll + текст-оверлей, скринкаст).
14. **Оптимизация хука**: K-кандидатов + отбор/скоринг.
15. **Per-platform пресеты**: прокинуть `aspectRatio` через `VideoStitch` (data-driven width/height) или удалить столбец; per-platform рерайт подписей.

### P3 — полировка
16. TTS-тиринг (`flash_v2_5` дефолт — дешёвый свитч сразу; `multilingual_v2`/`v3` для «геройского»).
17. Свериться с лицензией Remotion при 4+ сотрудниках (юр. обязательство).
18. Комплаенс/легал (EU AI Act маркировка, музыка, права на аватар).

## Decision points (нужно решение пользователя)
1. **Липсинк-движок:** Higgsfield Speak vs Sync.so vs HeyGen — A/B по русскому липсинку + себестоимость/ролик (разблокируется после P0.4 `VideoProvider`).
2. **Дистрибуция:** свой код на каждую площадку vs унифицированный постовщик (PostPeer/Upload-Post) — учесть аудит TikTok и ~15/день.
3. **Формат:** avatar-first vs multi-format (влияет на `video-storyboard` + Remotion-композиции).
4. **Рендер:** локальные реплики vs Remotion Lambda — порог миграции.
