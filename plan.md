# Контент-завод + SMM: расширенный план

## Сквозной слой: Brand Core (персонализация под бренд)

Это не отдельный этап, а слой, который пронизывает всё. Без него получится «ИИ-каша», неотличимая от конкурентов.

**Brand Knowledge Base** (БД, подаётся во все промпты и шаблоны):
- Tone of voice — 3–5 прилагательных + примеры «так / не так»
- Контент-пиллары: 3–5 тем, на которых стоит бренд
- Лексика: термины и слоганы (используем) vs стоп-слова и табу-темы
- Аудитория: 2–3 персоны (демография, боли, триггеры)
- Visual identity: палитра, шрифты, лого, отступы, шаблоны
- Competitor watchlist (5–10 брендов)
- Golden examples — эталонные посты, на которых учим модель

**Brand Checker** — отдельный LLM-агент, который перед публикацией сверяет контент с Knowledge Base и блокирует/правит несоответствия.

---

## Этап 1: Авто-мониторинг трендов

**Что делаем:** парсим TikTok Creative Center, Google Trends, YouTube Trending, X, Reddit, отраслевые RSS; мониторим, что публикуют конкуренты; слушаем упоминания бренда.

**Brand-фильтр:** каждый тренд получает score релевантности через LLM (на основе пилларов и табу). Низкий score → отброс.

**Output:** ежедневный дайджест из 5–10 топ-трендов с пометкой «почему это важно для нас».

**Стек:** Brandwatch / Mention / собственный парсер на Python + Apify; или готовое — Exploding Topics, Glimpse.

---

## Этап 2: Генерация идей и сценариев

**Что делаем:** LLM получает тренд + Brand Knowledge Base + формат, выдаёт 5–10 идей. По выбранной — полный сценарий: hook, body, CTA, подпись, хэштеги.

**Форматы под разные платформы:**
- Short-form video (15–60 сек): сценарий + раскадровка + текст под озвучку
- Carousel: 5–10 слайдов с текстом
- Stories: серия 3–5 кадров
- Long-form (статья / YouTube): структура + черновик

**Hooks Library** — банк проверенных зацепок под бренд, растёт со временем.

**Review queue:** человек подтверждает идею до монтажа. Без этого шага качество просядет — проверено.

---

## Этап 3: Автомонтаж / визуал

**Видео:**
- HeyGen / Synthesia — аватары и говорящие головы
- Runway / Pika — генеративные клипы
- Pictory / InVideo — монтаж из стоков по сценарию
- **Лучший вариант для брендирования:** свой pipeline на ffmpeg + Remotion (шаблоны на React, параметризуются) — полный контроль

**Изображения:**
- Midjourney / SD + LoRA, обученная на стиле бренда
- Canva API / Bannerbear — для шаблонных постов (цитаты, цены, новости)

**Brand Kit (обязательно):** intro/outro, водяной знак, шрифты, цветокор, lower-thirds, автосубтитры через Whisper в фирменном стиле.

---

## Этап 4: Автопостинг с расписанием

**API напрямую** (контроль): Meta Graph (FB + Instagram), TikTok Business, YouTube Data, Telegram Bot, X (платно), LinkedIn, VK.

**Готовые сервисы** (быстрый старт): Buffer, Later, Publer, Postiz (open-source), SMMplanner (для СНГ).

**Логика расписания:**
- Best time to post — авто по статистике аудитории
- Контент-микс — чтобы не было 5 reels подряд
- Evergreen-очередь — затыкает дыры в расписании
- A/B обложек и заголовков

---

## Этап 5: Интерфейс «несколько кнопок»

**Веб-приложение** (например, Next.js + Supabase + S3):
- Главная: «Тренды дня» → клик → «Создать контент»
- Кнопки: Перегенерировать / Утвердить / Запланировать / Опубликовать сейчас
- Календарь публикаций (drag & drop)
- Очередь модерации с ролями: автор → дизайнер → утверждающий
- История + поиск
- Слот для ручной загрузки

**MVP:** трёх кнопок хватит — «Сгенерировать», «Утвердить», «Опубликовать». Остальное наращивать по запросу команды.

---

## Отдельные задачи

**Автопостинг как отдельный инструмент.** По сути модуль из Этапа 4, вынесенный в самостоятельный сервис с API. И контент-завод, и ручные посты от команды должны ходить через него — иначе будете дважды строить интеграции с соцсетями.

**Дашборд метрик:**
- Сбор: те же API + Meta Insights, TikTok Analytics, YouTube Analytics, GA4
- Хранение: Postgres / ClickHouse
- Визуализация: Metabase / Looker Studio / свой фронт
- Метрики: охват, ER, прирост подписчиков, CTR, конверсии в целевое действие
- **Самое ценное** — атрибуция контента: какой тренд × формат × pillar → лучший результат. Эта обратная связь должна автоматически возвращаться в Этап 1–2 (учим систему выбирать лучше).

## Что бы я добавил поверх

**Roadmap по приоритетам:**
1. **MVP (4–6 нед):** Brand Knowledge Base + ручной выбор тренда + ИИ-сценарий + один шаблон монтажа + постинг в одну платформу
2. **v1 (2–3 мес):** автомониторинг + автомонтаж видео + постинг во все каналы + интерфейс
3. **v2:** Brand Checker, дашборд, A/B, автообучение на метриках

**Риски:**
- Лимиты и баны API соцсетей (особенно Instagram, TikTok)
- Очевидный ИИ-контент → теневой бан
- Авторские права на музыку, стоки, чужие тренды
- «Усреднение» бренда без сильного Brand Core

**Метрики успеха проекта:**
- Time-to-publish (тренд → пост): цель < 2 часов
- Стоимость одного поста vs агентство
- ER vs период до запуска
- Доля контента, одобренного с первой генерации (качество промптов)

Под этот стек я бы делал гибрид: TypeScript (Node.js) основной + Python для AI-воркеров. Это самый практичный вариант, и вот почему.
TypeScript / Node.js как основа — для API-сервера, автопостинга, шедулера, бэкенда веб-интерфейса:

Один язык с фронтом (Next.js из плана) — общие типы, общая команда, меньше переключения контекста
Remotion (программные видео-шаблоны, лучший вариант под брендирование) — это Node, на нём же отрисовка
Все соцсетевые SDK (Meta, TikTok, YouTube, Telegram, LinkedIn) первоклассно поддержаны в Node
I/O-bound нагрузка (а у вас 90% работы — это вызовы API) — это ровно то, в чём Node силён
BullMQ для очередей, Prisma для БД, NestJS или Fastify для структуры — всё зрелое

Python как воркеры — отдельные сервисы, которые принимают задачу из очереди и возвращают результат:

Скрейперы трендов (Scrapy, Playwright, BeautifulSoup) — здесь Python впереди
Whisper (транскрипция и субтитры) — нативный Python
Локальные ML-штуки: LoRA-инференс для картинок под бренд, классификаторы релевантности
Аналитика дашборда — pandas/polars удобнее SQL-only решений

Связь между ними — через очередь Kafka и REST
