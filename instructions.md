# Contento — инструкция пользователя

## Что это

Contento — контент-завод для SMM. Система мониторит тренды, генерирует идеи и сценарии через ИИ, проверяет их на соответствие бренду и публикует в соцсети по расписанию.

Основной цикл: **Тренд → Идея → Сценарий → Проверка → Публикация**.

---

## 1. Регистрация и вход

Откройте приложение. Если аккаунта нет — нажмите **Sign up**, введите имя, email и пароль (минимум 8 символов). Если аккаунт уже есть — **Sign in** с email и паролем.

---

## 2. Воркспейс

Весь контент живёт внутри воркспейса — отдельного пространства для бренда или проекта.

**Создание воркспейса:** нажмите на переключатель воркспейса в шапке → **New Workspace**. Заполните:
- **Name** — название (например, `Acme Brand`)
- **Slug** — латинские буквы, цифры и дефисы (например, `acme-brand`); должен быть уникальным

После создания воркспейс появится в выпадающем списке. Переключаться между воркспейсами можно в любой момент.

---

## 3. Brand Kit

**Навигация:** левое меню → **Brand Kit**

Brand Kit — база знаний о бренде. ИИ использует её при каждой генерации. Чем полнее заполнен кит, тем точнее контент. Настройте его **до** первой генерации.

Кит состоит из 7 вкладок:

### Voice & Tone
Тональность бренда. Добавьте 3–5 прилагательных.

Нажмите **Add**, введите:
- **Name** — прилагательное (например, `Дружелюбный`)
- **Description** — пояснение (необязательно)
- **Examples** — примеры через запятую (например, `«Привет!», «Рады видеть тебя»`)

Нажмите **Add** для сохранения. Удалить — кнопка **Delete** рядом с записью.

### Pillars
Контент-пиллары — 3–5 тем, на которых строится бренд.

Нажмите **Add**, введите:
- **Name** — название пиллара (например, `Инновации`)
- **Description** — описание (необязательно)
- **Keywords** — ключевые слова через запятую (например, `рост, технологии, будущее`)

### Vocabulary
Словарь бренда: разрешённые и запрещённые слова.

Нажмите **Add**, введите:
- **Word** — слово или фраза
- **Type** — выберите `ALLOW` (используем) или `FORBID` (никогда не используем)

### Personas
Портреты целевой аудитории. Добавьте 2–3 персоны.

Нажмите **Add**, введите:
- **Name** — название персоны (например, `Основатель стартапа`)
- **Description** — краткое описание (необязательно)
- **Pain Points** — боли через запятую (например, `нехватка времени, бюджет`)
- **Desires** — желания через запятую (например, `рост, признание`)

### Visual Identity
Визуальный стиль бренда. Заполните поля и нажмите **Save**:
- **Primary / Secondary / Accent Color** — цвета в формате `#rrggbb`
- **Primary / Secondary Font** — названия шрифтов
- **Logo URL** — прямая ссылка на логотип
- **Watermark URL** — прямая ссылка на водяной знак

### Competitors
Конкуренты для мониторинга.

Нажмите **Add**, введите:
- **Name** — название компании
- **URL** — сайт (необязательно)
- **Notes** — заметки (необязательно)

### Golden Examples
Эталонные посты — образцы, на которых ИИ учится стилю.

Нажмите **Add**, введите:
- **Title** — название примера
- **Content** — текст поста
- **Format** — формат (reel, carousel, story, text-post и др.)
- **Platform** — платформа (например, `Instagram`)

## 4. Подключение соцсетей

**Навигация:** левое меню → **Settings** → **Connected Accounts**

Без подключённых аккаунтов публикация невозможна. Подключите хотя бы один до начала работы.

Каждая платформа — отдельный раскрывающийся блок. Нажмите на название, заполните поля и нажмите **Connect**.

| Платформа | Что заполнить |
|-----------|--------------|
| **Telegram** | Bot Token, Channel ID (`@channel` или `-100xxx`), Display Name |
| **Instagram** | Access Token, IG User ID, Display Name |
| **TikTok** | Access Token, Open ID, Display Name |
| **YouTube** | Access Token, Refresh Token, Client ID, Client Secret, Display Name |
| **LinkedIn** | Access Token, Owner URN (`urn:li:person:xxx`), Display Name |
| **VK** | Access Token, Owner ID (отрицательное число для группы), Display Name |

Подключённые аккаунты отображаются списком вверху страницы. Удалить аккаунт — кнопка **Delete** рядом с ним.

## 5. Тренды

**Навигация:** левое меню → **Trends**

Здесь появляются тренды из автоматических источников (Google Trends, YouTube, Reddit, RSS) и добавленные вручную.

### Фильтрация
Кнопки вверху: **All / Pending / Analyzed / Archived** — фильтруют тренды по статусу.

### Действия над трендом
Каждый тренд отображается карточкой с цветным бейджем источника и скором релевантности (0–100).

- **Analyze** — ИИ анализирует тренд и выставляет relevance score на основе пилларов и словаря из Brand Kit. Статус меняется на `ANALYZED`.
- **Generate Ideas** — переходит на страницу Create Content с уже выбранным трендом (шаг 2).
- **Archive** — убирает тренд из активного списка.

---

## 6. Создание контента

**Навигация:** левое меню → **Create Content**

Страница разделена на две вкладки: **New Content** (основной флоу) и **Hooks Library**.

### New Content — 4 шага

#### Шаг 1: Enter Trend
Введите тренд вручную:
- **Title** *(обязательно)* — название тренда
- **Description** — описание (необязательно)
- **URL** — ссылка на источник (необязательно)

Нажмите **Next: Generate Ideas**. Тренд сохраняется в системе.

> Если пришли со страницы Trends через «Generate Ideas», шаг 1 пропускается автоматически.

#### Шаг 2: Pick Idea
ИИ генерирует 5–10 идей на основе тренда и Brand Kit. Каждая карточка содержит заголовок, угол подачи, формат и платформу.

Нажмите на карточку идеи, которая подходит. Можно вернуться назад кнопкой **Back**.

#### Шаг 3: Review Script
ИИ генерирует полный сценарий для выбранной идеи:
- **Hook** — зацепляющее начало
- **Body** — основная часть
- **CTA** — призыв к действию
- **Caption** — подпись для поста
- **Hashtags** — хэштеги

**Brand Check** — нажмите, чтобы запустить проверку сценария на соответствие Brand Kit. Результат показывает общий скор (0–100) и оценку по каждому критерию: tone, vocabulary, pillar, persona, visual, legal. При скоре < 70 появляются замечания.

Если Brand Checker предлагает **авто-фиксы** — нажмите **Apply Auto-Fixes**, чтобы применить исправления автоматически, или отредактируйте текст вручную.

**Render** — генерирует визуальный превью контента на основе Visual Identity из Brand Kit.

Нажмите **Approve** для перехода к шагу 4.

#### Шаг 4: Approve & Publish
Финальный шаг. Здесь можно:
- **Publish Now** — немедленная публикация в выбранный аккаунт
- **Schedule** — выбрать дату и время публикации
- **Submit for Review** — отправить в очередь на проверку (для команд с модерацией)

После действия нажмите **Start Over** для создания нового контента.

### Hooks Library
Банк успешных хуков. Хуки накапливаются автоматически из одобренных публикаций. Отображается текст хука, формат, источник и performance score.

---

## 7. Очередь проверки

**Навигация:** левое меню → **Review**

Сюда попадают сценарии, отправленные на проверку через «Submit for Review». Модератор видит hook, body, caption, brand score и дату отправки.

**Одобрить** — нажмите **Approve**. Сценарий уходит в статус `APPROVED`, готов к публикации.

**Отклонить** — нажмите **Reject**, откроется поле для комментария. Введите причину и нажмите **Submit Rejection**. Сценарий возвращается автору.

---

## 8. Календарь

**Навигация:** левое меню → **Calendar**

Показывает все сценарии и их публикации. Два режима отображения:

### List (список)
Сценарии сгруппированы по месяцам. Для каждого виден hook, платформа, формат, дата создания и статус (`DRAFT / BRAND_CHECKED / APPROVED / PUBLISHED`).

Нажмите **Publications** на карточке сценария — раскроется список публикаций с их статусами (`PENDING / PUBLISHING / PUBLISHED / FAILED`) и датами.

Фильтр по статусу — кнопки-таблетки вверху.

### Grid (сетка / календарь)
Месячный календарь. Публикации отображаются карточками на датах.

- Навигация по месяцам: кнопки **‹** и **›**
- **Drag & drop** — перетащите карточку публикации на другой день, чтобы перенести расписание. Изменение сохраняется автоматически.

---

## 9. Дашборд

**Навигация:** левое меню → **Dashboard**

Аналитика по воркспейсу. Требует выбранного воркспейса.

| Блок | Что показывает |
|------|---------------|
| **Funnel Stats** | Количество трендов → идей → сценариев → публикаций |
| **Publication Chart** | Публикации по платформам |
| **Attribution Table** | Какие платформы и форматы дают больше публикаций |
| **LLM Usage** | Расход токенов и стоимость по агентам и моделям |
| **Mention Alerts** | Срочные упоминания бренда (urgency ≥ 7) |

---

## 10. Управление командой

**Навигация:** левое меню → **Settings** → **Members**

Здесь видны все участники воркспейса с ролями. Владелец (OWNER) может менять роли участников и удалять их.

---

## Типичный рабочий день

```
1. Dashboard → проверить Mention Alerts
2. Trends → просмотреть новые тренды, нажать Analyze на интересных
3. Trends → нажать Generate Ideas на лучшем тренде
4. Create (шаги 2–4) → выбрать идею, получить сценарий, запустить Brand Check
5. Review → одобрить или отклонить входящие сценарии команды
6. Calendar (Grid) → скорректировать расписание на неделю
```

## Быстрый старт (минимум для первой публикации)

1. Создать воркспейс
2. Brand Kit → заполнить хотя бы Voice & Tone и 2–3 Pillars
3. Settings → Connected Accounts → подключить одну платформу
4. Create Content → пройти 4 шага → опубликовать
